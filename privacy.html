Privacy Policy

Last updated: 10 July 2026

Short version: Ledger doesn't have a server that receives, stores, or processes your personal data. It's a set of static pages that run in your browser. Your browser talks directly to a few third-party APIs to fetch public game data. No accounts, no cookies, no analytics, no tracking.

Who this policy is about

It is not a company or a registered business, so it does not publish a separate legal notice (Impressum) — that requirement generally applies to commercial or business-like online services, which this is not. What this site does not do

  • No account creation, login, or user profiles
  • No cookies
  • No analytics, telemetry, or advertising trackers of any kind
  • No server operated by this project that logs your visits or requests

The pages (index.html, ledger.html, item.html) are static files. When hosted, they're served as-is by the hosting provider (see "Hosting" below) — there is no application server behind them collecting information about you.

What your browser does when you use Ledger

Because there's no backend for the core app, your browser fetches data directly from a handful of third parties. Each of those services processes the request under its own privacy policy — Ledger never sees or stores this traffic.

These requests carry the same technical information any web request does (e.g. your IP address, visible to that third party as part of normal internet routing) but Ledger itself does not receive, log, or store any of it.

Local storage on your device

The Market Terminal caches crafting recipe data in your browser's localStorage so it doesn't have to re-fetch it every visit. This stays on your device — it is never transmitted to Ledger or any server we operate, and you can clear it at any time through your browser's site settings.

If you run the optional self-hosted backend

Ledger includes an optional backend (backend/ in the source repository) that you can run yourself in Docker to get sale history and cached seller name lookups. If you choose to run it:

  • It's your server, under your control — Ledger the project does not operate, host, or have access to it or any data it stores.
  • It logs basic request activity (address and requested path) to its own console output, standard for any web server, and stores that log only where you run it.
  • It resolves auction seller UUIDs to usernames via Mojang's public session server, so your browser never has to. That lookup happens server-to-server between your backend and Mojang, and results are cached in your own local database file.
  • None of this is transmitted back to the Ledger project in any form.
