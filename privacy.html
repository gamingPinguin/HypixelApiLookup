Privacy Policy

Last updated: 10 July 2026

Short version: the Site has no server that receives, stores, or processes personal data. It consists of static pages that run in the visitor's browser, which communicates directly with a small number of third-party APIs to fetch public game data. No accounts, no cookies, no analytics, no tracking.

What the Site does not do

  • No account creation, login, or user profiles
  • No cookies
  • No analytics, telemetry, or advertising trackers of any kind
  • No server operated by this project that logs visits or requests

The pages (index.html, bazaar-tracker.html, auction-tracker.html, flips.html) are static files. Once hosted, they are served as-is by the hosting provider (see "Hosting" below); no application server sits behind them collecting visitor information.

What the browser does when visiting the Site

Because there is no backend for the core application, the visitor's browser fetches data directly from a handful of third parties. Each of those services processes the request under its own privacy policy; the Site itself never receives or stores this traffic.

These requests carry the same technical information any web request does (for example, an IP address, visible to that third party as part of normal internet routing), but the Site does not receive, log, or store any of it.

Local storage on the visitor's device

The Flips page caches crafting recipe data in the browser's localStorage so it does not need to be re-fetched on every visit. This data stays on the visitor's own device — it is never transmitted to the Site or any server the project operates, and can be cleared at any time through the browser's site settings.

Optional self-hosted backend

Ledger includes an optional backend (backend/ in the source repository) that can be self-hosted via Docker to enable sale history and cached seller name lookups. Where that backend is run:

  • It is operated entirely by whoever deploys it — the Ledger project does not operate, host, or have access to it or any data it stores.
  • It logs basic request activity (address and requested path) to its own console output, standard for any web server, stored only wherever it is deployed.
  • It resolves auction seller UUIDs to usernames via Mojang's public session server, so the browser does not have to. That lookup happens server-to-server between the backend and Mojang, with results cached in a local database file under the deployer's control.
  • None of this is transmitted back to the Ledger project in any form.

Hosting

Ledger is intended to be published on GitHub Pages. GitHub, Inc. (or GitHub B.V. for users in the EEA/UK/Switzerland) serves the static files and, like any web host, processes standard access logs (e.g. IP address, requested file, timestamp) as part of operating that infrastructure. That processing is governed by GitHub's own Privacy Statement, not by this project — the Site does not receive a copy of these logs.
